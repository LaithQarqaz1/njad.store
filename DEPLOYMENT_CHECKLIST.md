## 0) الأسرار — إلزامي قبل أول نشر (يدوي)

القيمة السابقة لـ `GW_ADMIN_SECRET` كانت مكتوبة نصاً داخل `backend/wrangler.toml` وملتزمة في git → **تعتبر مكشوفة ويجب تدويرها**، وقد أزيلت من الملف. بدون هذه الخطوة سيفقد الـ worker قيمة SMS gateway admin بعد النشر التالي:

```bash
# من داخل backend/ — أدخل قيمة جديدة قوية (لا تعيد استخدام القديمة):
npx wrangler secret put GW_ADMIN_SECRET
npx wrangler secret put SMS_GATEWAY_MASTER_PASSWORD
```

- [ ] توليد قيمة جديدة لـ `GW_ADMIN_SECRET` وضبطها كسر على **worker المتجر**.
- [ ] ضبط **نفس القيمة الجديدة** على worker الدليل المنفصل (`sms.laithstoreservices.workers.dev`).
- [ ] ضبط `SMS_GATEWAY_MASTER_PASSWORD` (كلمة المرور الثابتة لإدارة بوابات SMS من اللوحة).

## 1) نشر الـ worker الرئيسي

```bash
cd backend
npx wrangler deploy
```

يفعّل دفعة واحدة: إصلاح دوامة الجلسات (reuse/adopt)، ردّ sync الغني، إصلاح 409 لإجراءات الأدمن، client-levels من D1، أسعار الواجهة حسب المستوى من D1، إجمالي الأرصدة من Neon، إعادة محاولة `neon_timeout`، تصنيف provider-timeout الصحيح، إصلاحات هذا الفرع (OTP المشفر، account-info من D1، upsertUser الذرّي، حظر الدخول D1، عدم تسريب الأخطاء، اجتياح الطلبات القديمة).

- [ ] `npx wrangler deploy` نجح.
- [ ] `npx wrangler tail` مفتوح أثناء التحقق (الخطوة 6).

## 2) نشر الواجهة (Cloudflare Pages / الاستضافة الثابتة)

الملفات التي يجب أن تصل للاستضافة بآخر نسخة:

- [ ] `site-bundle.js` (الأثر الفعلي المُحمّل)
- [ ] `index.html` (يحمل cache-bust الحالي `?v=20260701-appname-onerequest-03` — تأكد أن Network يحمّل هذا الإصدار بعد النشر)
- [ ] `_headers` (سياسة الكاش: أصول مؤرَّخة long-cache + `no-store` لـ index.html/sw.js/site_settings.js)
- [ ] تحقّق في المتصفح: `site-bundle.js?v=` يقرأ الإصدار الجديد (وليس نسخة قديمة من الكاش).

## 3) إعادة تشغيل ترحيل المستخدمين إلى D1 (زر لوحة المطور)

بعد نشر الـ worker: **لوحة الأدمن → قسم المطور → «ترحيل المستخدمين إلى D1»** (الزر يكرر تلقائياً حتى النهاية).

يزرع: جلسات كل المستخدمين (hash)، `total_spent_mills` (بدونه يبدأ إجمالي الصرف من 0)، توكنات API الموجودة (بدونها تتعطل تكاملات API القديمة حتى يعاد توليدها)، وأرصدة العرض `balance_mills` من Neon عبر `listWalletBalances` (بدونها يعرض الأدمن/البوت أرصدة صفرية حتى أول عملية مالية).

- [ ] تشغيل الزر حتى ظهور "done".
- [ ] التحقق من العدادات (عدد المستخدمين المرحّلين = المتوقع).

## 4) تطبيق / تحديث قواعد Neon (زر لوحة المطور)

**لوحة الأدمن → قسم المطور → «تطبيق / تحديث قواعد Neon»** (idempotent — آمن التكرار).

يضمن أن الإنتاج يحمل: `fin_create_order` بدون بوابة الطلب-الواحد (سماح بطلبات متزامنة)، دوال `fin_refund_captured_order` / `fin_recapture_refunded_order`، و`fin_redeem_code`.

- [ ] تشغيل الزر والتحقق من عدم وجود عبارات فاشلة في النتيجة.

## 5) منظومة SMS (إن كانت مفعّلة)

- [ ] نشر worker الدليل المنفصل (مشروع `sms-gateway`) بأسراره.
- [ ] من لوحة الأدمن: تسجيل البوابة (يتطلب `GW_ADMIN_SECRET` الجديد على الطرفين + `SMS_GATEWAY_MASTER_PASSWORD`).
- [ ] تهيئة تطبيق الأندرويد بتسجيل الدخول للخادم.

## 6) التحقق بعد النشر

- [ ] **تسجيل الدخول → صفحة الإعدادات**: تُحمَّل من أول مرة (اسم/هاتف/مستوى/إجمالي صرف ليست `--`؛ إجمالي الصرف ليس 0 لمستخدم قديم).
- [ ] **ثبات sessionKey**: أعد تحميل الإعدادات مرتين — نفس `sessionKey` في الردّين (لا سكّ جديد كل مرة).
- [ ] **الشراء → الرصيد**: نفّذ شراءً؛ الرصيد في الهيدر يتحدث فوراً بالقيمة الصحيحة.
- [ ] **الإيداع**: إيداع تجريبي (يدوي أو USDT) يصل ويُقيّد.
- [ ] **حظر D1**: احظر حساب اختبار من اللوحة → محاولة تسجيل دخوله تُرفض بـ `auth/banned` (وليس دخولاً ناجحاً يليه 403).
- [ ] **مراقبة `wrangler tail`**: لا `users_d1_write_failed`، لا أخطاء `verifyUserSession` متكررة.
- [ ] **قبل تفعيل `CATALOG_D1_PURCHASE="1"`**: راقب `catalog_d1_purchase_identity_mismatch` / `_qty_mismatch` / `_row_missing` في tail حتى تصفو، ثم اقلب العلم وأعد النشر.

## 7) أعلام بيئة اختيارية جديدة (موثقة في wrangler.toml)

| العلم | الافتراضي | الوظيفة |
|---|---|---|
| `ORDER_STALE_AUTO_ACCEPT_DAYS` | `3` | عمر الطلب المعلق قبل قبوله تلقائياً وتنبيه الأدمن (بأزرار تراجع). `"0"` يعطّل الاجتياح. |
| `BAN_FIRESTORE_MIRROR` | معطّل | `"1"` يعيد كتابة مرآة الحظر في Firestore رغم القطع (نسخة دائمة اختيارية). |

## ملاحظات تراجع سريعة

- `USERS_D1_SOURCE="0"` + إعادة نشر: يعيد قراءة المستخدمين/الجلسات إلى المسار القديم.
- `ORDER_STALE_AUTO_ACCEPT_DAYS="0"`: يوقف القبول التلقائي للطلبات القديمة فوراً.
- `SITE_STATE_D1` / أعلام `CATALOG_D1_*`: قابلة للقلب فورياً كما هو موثق في wrangler.toml.
